## 各引擎职责

### 数据引擎 — 数据供给层
- 全市场行情拉取（Tencent/AKShare/BaoStock 三级降级）
- DuckDB 持久化（快照、日线、历史）
- 公司概况管理
- 独立 REST API (`/api/v1/data/*`)
- **设计目标：可被第三方替换，只要接口契约一致**

### 聚类引擎 — 基本面
- BGE 语义嵌入 + HDBSCAN 聚类 + UMAP 降维
- 3D 地形插值
- 基本面特征（PE/PB/市值/换手率等）
- 多归属软隶属度
- 依赖数据引擎获取原始数据

### 信息引擎 — 消息面（待建设）
- 新闻/公告数据源接入
- NLP 情感分析（利好/利空/中性）
- 事件提取（并购、增减持、业绩预告等）
- 行业/个股舆情热度
- 独立 REST API (`/api/v1/news/*`)

### 量化引擎 — 技术面（待增强）
- 技术指标计算（MA/RSI/MACD/布林带等）
- 多因子模型（当前 13 因子，可扩展）
- IC/ICIR 因子回测
- 量化信号生成
- 独立 REST API (`/api/v1/quant/*`)

### 专家层 — AI 分析师
- MCP Server 提供工具访问各引擎
- LLM 综合三个面（基本面+消息面+技术面）进行分析
- 支持两种交互方式：
  - Agent 对话（Claude Code / MCP Client）
  - 前端对话面板（需 LLM API 支持）
- RAG 增强（远期）：历史分析报告、研报知识库

## 实施阶段

### Phase 1：数据引擎 / 聚类引擎分离 ← 当前
- 状态：spec 已完成，待实施
- 详见：`2026-03-14-data-cluster-engine-separation-design.md`
- 交付物：`data_engine/` + `cluster_engine/` 模块化

### Phase 2：量化引擎独立
- 将 `predictor_v2` + `factor_backtest` 从聚类引擎中提取为独立模块
- 扩展技术指标计算（当前只有 6 个 COMPUTED_FEATURES）
- 独立 MCP tools（当前混在 10 个 tool 里）
- 前提：Phase 1 完成

### Phase 3：信息引擎建设
- 选择新闻数据源（财联社/东方财富/新浪财经）
- 情感分析模型（可用 LLM 或轻量 NLP 模型）
- DuckDB 新增新闻/公告表
- 新增 MCP tools（查新闻、查舆情）
- 前提：Phase 1 完成（数据引擎接口稳定）

### Phase 4：专家层整合
- MCP Server 扩展：跨引擎组合工具
- 结构化分析模板（基本面+消息面+技术面三维报告）
- 前端对话面板接入 LLM API
- 前提：Phase 2 + Phase 3 完成

### Phase 5：RAG 增强（远期）
- 历史分析报告向量化存储
- 研报/公告知识库
- 检索增强生成
- 前提：Phase 4 完成

## 设计约束

- 各引擎通过 Python 接口通信（同进程），未来可拆为微服务
- 每个引擎有自己的 schemas、routes、核心逻辑
- 数据引擎是共享基础设施，其他引擎都依赖它
- MCP Server 是各引擎对外的 AI 接口，保持 stdio transport
- 新增引擎不破坏现有 API 路径
