# 独立行业知识引擎（IndustryEngine）— 长期 Roadmap

> 当前辩论系统已实现 LLM 生成+缓存的行业认知层（IndustryCognition），作为过渡方案。长期需要真实数据源支撑。

## 现状

- 辩论只支持单只股票，不支持行业/板块级辩论
- 行业认知由 LLM 实时生成，缓存在 DuckDB（按行业+日期去重）
- 认知内容：产业链上下游、成本结构、壁垒、供需格局、认知陷阱、周期定位、催化剂/风险

## 长期目标：IndustryEngine

新建 `engine/industry_engine/`，类似 InfoEngine 架构：

1. 预构建行业知识图谱（产业链关系、上下游映射）
2. 接入行业级数据源（板块行情、行业资金流向、行业新闻）
3. 提供 `get_industry_analysis(industry)` 等 API
4. 用 LLM 缓存数据作为冷启动种子，逐步用真实数据源替换

## 扩展方向：行业级辩论

- 辩论入口支持行业/板块主题（如"油气"、"化工"）
- 自动选取行业代表性个股作为辩论锚点
- 行业级 facts 拉取（板块涨跌、龙头表现、资金流向）

## 状态

待启动。等 LLM 生成行业认知积累到一定量级后考虑。
