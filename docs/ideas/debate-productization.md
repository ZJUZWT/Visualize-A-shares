# 辩论系统产品化方向 — 待讨论

> 核心问题：辩论是分析工具，但用户要的是决策工具。多空辩论完，用户还是不知道该怎么操作持仓。

## 候选方向

### 1. 持仓管理器
用户录入自己的持仓，系统针对每只持仓股给出"持有/减仓/加仓/清仓"建议，辩论作为建议的论据支撑。

### 2. 信号看板
把辩论结果量化成信号（强烈看多/温和看多/中性/温和看空/强烈看空），配合因子评分和技术指标，做成仪表盘。

### 3. 投资专家主动推送
让 expert agent 主动监控用户关注的股票，发现异动时主动发起辩论并推送结论。

### 4. 组合优化
用户给出资金量和风险偏好，系统基于辩论结果 + 因子评分 + 相关性，输出推荐组合配比。

## 状态
待讨论，未确定方向。

## 辩论深度优化 — 多维度视角 vs 多 Agent

### 问题
单一多头/空头视角太单薄，但加角色会导致 token 和调用数量激增。

### 结论
不加角色，加视角。在现有 bull/bear 的 system prompt 里要求从多个维度论证：
- 多头：基本面 + 技术面 + 事件催化
- 空头：估值风险 + 技术破位 + 政策/黑天鹅

用 prompt engineering 替代 multi-agent，效果接近但成本不变。
